# Contributing to Laravel DDD Starter

Thank you for considering contributing to Laravel DDD Starter! We appreciate your interest in improving this package.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature: `git checkout -b feature/my-feature`
4. Make your changes
5. Run tests to ensure everything works
6. Commit your changes with clear messages
7. Push to your fork
8. Submit a pull request

## Code Standards

- Follow PSR-12 coding standards
- Use meaningful variable and function names
- Add type hints where possible
- Write descriptive comments for complex logic

## Testing

Before submitting a PR, ensure:

1. All tests pass: `composer test`
2. Code style is correct: `composer lint`
3. No deprecation warnings

## Reporting Issues

When reporting issues, please include:

1. Laravel version
2. PHP version
3. Steps to reproduce
4. Expected behavior
5. Actual behavior
6. Error message or log output

## Feature Requests

Feature requests are welcome! Please:

1. Describe the feature clearly
2. Explain the use case
3. Provide examples if possible
4. Discuss potential implementation

## Documentation

If adding a new feature:

1. Update relevant documentation
2. Add code examples
3. Update CHANGELOG.md
4. Add entry to docs/commands.md if it's a command

## Pull Request Process

1. Update documentation and comments
2. Add tests for new functionality
3. Ensure all tests pass
4. Update CHANGELOG.md
5. Fill out the PR template completely
6. Reference any related issues

## Areas for Contribution

- Bug fixes
- Documentation improvements
- New commands
- Performance optimizations
- Tests
- Code quality improvements
